Nmap Network Discovery – Seeing the Matrix

Nmap Recon Guide: How to See the Machines Around You

If you're on a Red Team mission — even a simulated one — your first task is simple: see the invisible. Before touching a port or probing a service, you need to know what's around. Here's how to do that with one of Kali Linux's most powerful tools: nmap.

🎯 Objective: Discover All Active Devices on a Network

Imagine you're dropped into a network with no map, no beacon, just your instincts and tools. You don’t know who's there — but you need to.

Nmap can scan your local environment and list all live machines: laptops, smartphones, routers, IoT devices, you name it.

🧠 Step 1 – Know Your Own IP and Subnet

ip a

Look for a line like:

inet 192.168.1.23/24

This means your IP is 192.168.1.23 and your subnet is /24, aka 192.168.1.0 - 192.168.1.255.

📡 Understanding Hosts, Ports & Services

🧍 Host

A host is any machine connected to the network — a PC, a smartphone, a printer, or even a smart lightbulb. Each host has an IP address on the network (like 192.168.1.42).

🔌 Ports

A port is like a virtual door on a host. It's where services run and communicate. Ports are numbered from 0 to 65535. Some common ones:

  • 22 – SSH (remote login)
  • 80 – HTTP (web server)
  • 443 – HTTPS (secure web)
  • 21 – FTP (file transfer)
  • 3389 – RDP (Windows remote desktop)

Ports can be:

  • Open – service is listening
  • Closed – nothing is there
  • Filtered – blocked by firewall or security tool

⚙️ Services

A service is the actual software running behind a port. For example:

  • Port 22 might run OpenSSH
  • Port 80 could be Apache or Nginx

Nmap doesn't just detect ports — it can identify the service and sometimes its version (Apache 2.4.41 for example).

🛰️ Step 2 – Ping Scan the Whole Network

nmap -sn 192.168.1.0/24

This will send ping requests to all 254 possible hosts. You'll get a list of machines that respond.

📦 Output Example

Nmap scan report for 192.168.1.1 (Router)
Host is up.
Nmap scan report for 192.168.1.42 (Ubuntu-MSI)
Host is up.
Nmap scan report for 192.168.1.74
Host is up.

🧩 Step 3 – Scan Detected Hosts One by One

Once you identify IPs, scan their ports and services:

nmap -sV -O 192.168.1.42

This reveals what the device is running — HTTP server, SSH, Samba, RDP, etc. Even OS fingerprinting.

📂 Optional: Save Results for Logs

nmap -sn 192.168.1.0/24 -oN scan_results.txt

You can integrate the output into HyugaLog or a Red Team report later.

🛡️ Use Case – Red Team Simulation

  • You've breached a network but need passive intel first
  • You're training OSINT and need to map a lab network
  • You want to alert a team on how visible their devices are

⚠️ Ethics Reminder

Never scan networks you don’t own or have explicit permission to test.

Nmap is powerful, but with great power comes operational responsibility.

📌 Final Note

Scanning is not just technical. It's psychological. You're not looking at machines — you're reading a system's posture, assumptions, weaknesses. You're entering someone else's territory. Map it well.
